Always use human-friendly formatting when displaying date and/or time.

Date In Progress

Format Examples
[month (mixed case 3 letter abbreviation)] [numerical day], [year (4 digit)] Nov 28, 2016
Apr 2, 2016
Use an en dash (&ndash) to separate the dates in date ranges. Nov 28, 2016–Nov 30, 2016

Time In Progress

Best Practices:

  • Timezone control flow: Always convert to local time based user preference. If unavailable, use assumed time based on browser. Only use UTC when no timezone preference can be determined or standardization across timezones is necessary.
  • Alway convert to local time based user preference. If unavailable, use assumed time based on browser. Only use UTC when no timezone preference can be determined or standardization across timezones is necessary.
  • Always offer users the ability to convert to UTC
  • Add UTC offset to timezone only when that specificity is important. E.g “CST (UTC-6)”
  • Add seconds or milliseconds to the time only when that level specificity is important
Format Examples
[time (12 hour clock)] [AM | PM (capitalized)] 7:58 PM CST
[time (12 hour clock)] [timezone (capitalized 3 letter abbreviation)] 12:59:16 AM CDT (UTC-5)

TimeagoIn Progress

Timeago details length of time that has elapsed since an event E.g. a monitoring alarm or ticket update. This is helpful for orienting users to the timeline of events.

Best Practices:

  • Display order:
    1. Under 1 minute: “just now”
    2. After first minute: “1 min”
    3. After second minute: “[#] mins”
    4. After first hour: “1 hr”
    5. After second hour: “[#] hrs”
    6. After 24 hours: “1 day”
    7. After 48 hours: “[month (mixed case 3 letter abbreviation)] [numerical day]” e.g “Nov 3”
  • Hover tooltip with “[timestamp]”
